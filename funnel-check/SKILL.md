# 漏斗健康檢查

Claude Code 自動抓漏斗數據、算轉換率、找出問題、給具體修法。學員不需要打開任何 Dashboard。

## 使用方式

`/funnel-check` — Claude Code 自動執行完整檢查

---

## 執行流程（Claude Code 自動處理）

### Step 1：抓數據

Claude Code 自動執行：
```bash
curl -s https://學員網域/funnel
curl -s https://學員網域/track/stats
```

如果沒有網域資訊，問學員一個問題：
**問學員：** 你的網域是什麼？

### Step 2：計算轉換率

Claude Code 自動計算每一關的轉換率，對照健康指標：

| 環節 | 健康 | 警戒 | 危險 |
|------|------|------|------|
| 廣告 CTR | > 2% | 1-2% | < 1% |
| LP > 加 LINE | > 20% | 10-20% | < 10% |
| 好友 > 報名 | > 60% | 40-60% | < 40% |
| 報名 > 到課 | > 40% | 30-40% | < 30% |
| 到課 > 完課(777) | > 70% | 50-70% | < 50% |
| 完課 > 意願 | > 15% | 10-15% | < 10% |
| 意願 > 成交 | > 30% | 20-30% | < 20% |

### Step 3：找出問題 + 給修法

Claude Code 自動找出最差的環節，給具體的修改建議：

**廣告 CTR 低（< 1%）：**
- 換素材（Hook 前 3 秒沒抓住注意力）
- 換受眾（投錯人了）
- 行動：用 `/ai-writer` 產新素材方向

**LP > 加 LINE 低（< 10%）：**
- CTA 不夠明顯
- LP 太長，人還沒滑到就走了
- 行動：把 CTA 往上移 + 強化第一屏的痛點

**報名 > 到課率低（< 30%）：**
- 確認 4 次提醒有發（24h/6h/2h/10m）
- 行動：Claude Code 自動檢查 Bot 的提醒排程

**到課 > 完課(777) 低（< 50%）：**
- 直播太無聊或沒互動
- 777 喊的時機不對（應在報價前）
- 行動：用 `/ai-writer 講綱` 重新設計互動節奏

**完課 > 意願低（< 10%）：**
- 報價太高沒有錨定
- 沒有社會證明
- 行動：用 `/ai-writer 講綱` 重新設計報價段落

**意願 > 成交低（< 20%）：**
- 追單文案沒有力或沒在發
- 行動：用 `/ai-writer 追單` 重新產文案

### Step 4：輸出報告

Claude Code 自動輸出格式化報告，標記每個環節的狀態（健康/警戒/危險），並列出「本週最該改的一件事」。

## 漏斗結構

```
Meta 廣告曝光
  > CTR（點擊率）
LP 瀏覽
  > 加 LINE 率
LINE 好友
  > 報名率
報名直播
  > 到課率
到課
  > 完課率（777）
完課
  > 意願表達率
表達意願
  > 成交率
付款成交
```

## CPL 告警

Claude Code 自動判斷：
- CPL < $50 — 很好，可以加預算
- CPL $50-100 — 還行，觀察
- CPL > $100 — 建議換素材或暫停

## 每週 SOP

Claude Code 自動提醒：
1. 找出最低的轉換率
2. 對照問題表，選一個改動
3. 改完這週不再動
4. 下週再看有沒有改善

**一次只改一個。改完等一週。**
