# 漏斗健康檢查

檢查漏斗每一關的轉換率，告訴你哪裡有問題、怎麼修。

## 使用方式

`/funnel-check` — 檢查漏斗數據 + 給優化建議

## 行為

1. 抓 Dashboard 數據：`curl -s https://你的網域/dashboard` 或 `/funnel`
2. 計算每一關轉換率
3. 對照問題表，找出最該優化的環節
4. 給具體的修改建議

## 漏斗結構

```
Meta 廣告曝光
  ↓ CTR（點擊率）
LP 瀏覽
  ↓ 加 LINE 率
LINE 好友
  ↓ 報名率
報名直播
  ↓ 到課率
到課
  ↓ 完課率（777）
完課
  ↓ 意願表達率
表達意願
  ↓ 成交率
付款成交
```

## 健康指標

| 環節 | 健康值 | 警戒值 | 危險值 |
|------|--------|--------|--------|
| 廣告 CTR | > 2% | 1-2% | < 1% |
| LP → 加 LINE | > 20% | 10-20% | < 10% |
| 好友 → 報名 | > 60% | 40-60% | < 40% |
| 報名 → 到課 | > 40% | 30-40% | < 30% |
| 到課 → 完課(777) | > 70% | 50-70% | < 50% |
| 完課 → 意願 | > 15% | 10-15% | < 10% |
| 意願 → 成交 | > 30% | 20-30% | < 20% |

## 數字不好 → 改什麼

### 廣告 CTR 低（< 1%）
- 換素材（Hook 前 3 秒沒抓住注意力）
- 換受眾（投錯人了）
- 圖片/影片太普通，沒有停下來看的理由
- **行動：** 用 `/ai-writer` 產新素材方向

### LP → 加 LINE 低（< 10%）
- CTA 不夠明顯
- LP 太長，人還沒滑到就走了
- 痛點沒打中
- **行動：** 把 CTA 往上移 + 強化第一屏的痛點

### 報名 → 到課率低（< 30%）
- 提醒不夠 → 確認 4 次提醒有發（24h/6h/2h/10m）
- 提醒太無聊 → 換提醒文案
- 直播時間不好 → 試不同時段
- **行動：** 檢查 Bot 的提醒排程有沒有正常跑

### 到課 → 完課(777) 低（< 50%）
- 直播太無聊，人中途走了
- 沒有互動，觀眾滑手機
- 777 喊的時機不對（應在報價前）
- **行動：** 檢查直播講綱的互動節奏

### 完課 → 意願低（< 10%）
- 報價太高沒有錨定
- 沒有社會證明
- CTA 不清楚
- **行動：** 用 `/ai-writer 講綱` 重新設計報價段落

### 意願 → 成交低（< 20%）
- 追單文案沒有力
- 追單沒在發（系統問題）
- 異議沒處理（太貴/沒時間/再想想）
- **行動：** 用 `/ai-writer 追單` 重新產文案

## 每週 SOP

```
週一：打開 Dashboard
  ↓
找出最低的轉換率
  ↓
對照上面的表，選一個改動
  ↓
改完，這週不再動
  ↓
下週一看有沒有改善
```

**一次只改一個。改完等一週。**

## CPL 告警

每次 Lead 成本（CPL）超過設定值自動通知：

```
CPL < $50 → 🟢 很好，可以加預算
CPL $50-100 → 🟡 還行，觀察
CPL > $100 → 🔴 換素材或暫停
```
